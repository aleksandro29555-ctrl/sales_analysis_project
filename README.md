# Sales Analysis Pipeline (Airflow)

## Описание

Проект реализует автоматизированный анализ пользовательских действий в интернет-магазине.
Цель — определить популярные товары, поведение пользователей и факторы, влияющие на покупку.

## Что делает DAG

Airflow DAG выполняет 3 шага:
	1.	Загружает данные из CSV в SQLite
	2.	Анализирует поведение пользователей и рассчитывает метрики
	3.	Сохраняет результаты и пишет лог об успешном выполнении

## Метрики
	•	Топ товаров по просмотрам, добавлениям в корзину и покупкам
	•	Выручка по категориям
	•	Конверсия: просмотр → корзина → покупка

## Результаты
	•	Excel-файл с метриками
	•	Интерактивные графики (Plotly, HTML)

## Стек
	•	Python (pandas, plotly)
	•	SQLite
	•	Apache Airflow (Docker)

## Запуск
```docker compose up -d```
Airflow UI: http://localhost:8080
Login: airflow / airflow

## Структура проекта
	•	dags/ — DAG Airflow
	•	scripts/ — скрипты загрузки и анализа
	•	data/ — исходные данные
	•	output/ — результаты анализа

## Результат

Автоматизированный pipeline, который позволяет регулярно анализировать поведение пользователей и получать готовые метрики и визуализации.
